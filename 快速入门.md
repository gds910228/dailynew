# 快速入门 - 5分钟上手

## 🎯 你现在需要做的（按顺序）

### ⚠️ 重要：先解决小程序无内容问题

你的数据文件已经在GitHub上了，代码已配置多个备用数据源，会自动尝试连接。

**立即解决：**

1. **确保域名校验已关闭（开发阶段必须）**
   - 微信开发者工具 → 详情 → 本地设置
   - 勾选 ✅ 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书

2. **清除小程序缓存并重新编译**
   - 微信开发者工具 → 项目 → 清除缓存
   - 点击"编译"按钮

3. **查看Console日志**
   - 微信开发者工具 → Console标签
   - 查找"尝试加载源"日志
   - 代码会自动尝试3个数据源：
     - GitHub API
     - Raw GitHub
     - jsDelivr CDN

4. **仍然无法加载？**
   - 查看 **网络问题排查指南.md** 获取详细解决方案
   - 测试浏览器能否访问：https://api.github.com/repos/gds910228/dailynew/contents/data/articles.json
   - 尝试使用手机热点测试网络连接

## 📅 每日发布新文章的流程

### 方法1：最简单（推荐新手）

#### 第1步：准备图片
```
1. 将你的图片文件重命名为：20250119001.jpg（日期+序号）
2. 复制到：assets/images/ 目录
```

#### 第2步：上传图片到GitHub
```
打开Git Bash或命令行，执行：

cd D:\WorkProjects\AI\MiniProgram\dailynew
git add assets/images/
git commit -m "添加图片"
git push origin main
```

#### 第3步：获取图片URL
图片URL格式：
```
https://raw.githubusercontent.com/gds910228/dailynew/main/assets/images/20250119001.jpg
```

#### 第4步：打开Web后台
双击运行：**`open-admin.bat`**

#### 第5步：填写并提交
1. 填写文章信息
2. 粘贴图片URL
3. 点击"提交文章"

#### 第6步：验证
小程序下拉刷新，查看新文章

---

### 方法2：使用自动化脚本（推荐）

#### 超简单操作：

1. **双击运行** `快速发布.bat`
2. **将图片复制到** `assets/images/` 目录
3. **在命令行输入** `y` 确认
4. **脚本会自动**：
   - 上传图片到GitHub
   - 生成图片URL列表
   - 打开Web管理后台
5. **复制URL到Web后台** → 填写信息 → 提交

**就这么简单！**

## ⚡ 一键操作流程

### 日常维护一键命令

创建一个批处理文件 `每日发布.bat`：

```batch
@echo off
echo.
echo ========================================
echo 每日发布流程
echo ========================================
echo.

echo [1/4] 上传图片到GitHub...
git add assets/images/
git commit -m "添加图片"
git push origin main

echo.
echo [2/4] 生成图片URL...
python -c "import os; files=os.listdir('assets/images'); [print(f'https://raw.githubusercontent.com/gds910228/dailynew/main/assets/images/{f}') for f in files if f.endswith(('.jpg','.png'))]"

echo.
echo [3/4] 打开Web管理后台...
start admin/index.html

echo.
echo [4/4] 完成！
echo 请在Web后台填写文章信息并提交
echo.
pause
```

**每天只需双击这个文件！**

## 🔧 常见问题速查

| 问题 | 解决方法 |
|------|---------|
| 小程序无内容 | 检查域名白名单或启用"不校验合法域名" |
| 图片不显示 | 检查URL是否正确，在浏览器测试URL |
| 无法提交到GitHub | 检查Token是否正确，网络是否正常 |
| 找不到assets/images目录 | 脚本会自动创建，或手动创建 |

## 💡 专业提示

### 图片优化
使用 TinyPNG 压缩图片：https://tinypng.com/
可以节省50-80%的流量

### 批量处理
如果要发布多篇文章，可以：
1. 一次性上传所有图片
2. 批量生成URL列表
3. 依次在Web后台提交

### 定期备份
每月备份一次数据文件：
```bash
cp data/articles.json data/backup/articles_$(date +%Y%m%d).json
```
